# Cap Table Timeline View

*This article is for founders and admins using Fairmint Studio.*

The Timeline View shows every equity event in your company's history, organized chronologically — from incorporation to the latest issuance. This is where you **upload documents**, **review AI-extracted data**, **edit individual events**, and **commit changes** to your live cap table.

Go to **Cap Table > Timeline View** in the left menu.

> **Note**: Timeline View is currently available to super-admins only.

---

## Page Layout

The page has three main areas:

1. **Header** — page title ("Cap Table Timeline"), subtitle ("Build & Visualize your company's equity story"), and action buttons (**Revert**, **Apply**, **+ Add**, and optionally **Cap Table AI**)
2. **Metrics row** — six key statistics: Amount Raised, Stakeholders, ESOP Shares, Valuation, Share Price, and Authorized Shares
3. **Timeline** — a vertical chronological list of all equity events, grouped by date

---

## Getting Started: Upload Documents

If your cap table is empty, you'll see a welcome screen prompting you to upload files. Click **Choose files to upload**.

If your cap table already has data, click **+ Add** in the top-right corner to upload more files.

### What to Upload

The system accepts any documents related to your company's equity history:

- Stock purchase agreements
- SAFEs & convertible notes
- Board consents & resolutions
- Option grant letters
- Cap table exports (from Carta, Pulley, AngelList, etc.)
- 409A valuations

You can upload multiple files at once (up to **50 MB per file**). Accepted formats include PDF, DOCX, XLSX, TXT, and common image formats (PNG, JPEG, WebP, TIFF). ZIP files are not supported.

For more details, see [Import Your Cap Table](3.1-upload-import-cap-table.md).

### What Happens After Upload

After uploading, Fairmint's AI processes each document through several stages — extraction, classification, data creation, and validation. You don't need to track each stage; the page shows overall progress.

The key milestone is **classification**: once the AI identifies each document type, you'll be asked to review the results before continuing. During review, you can:

- **Edit classification tags** — add or remove tags to correct the document type. Tags come from a predefined list.
- **Adjust the date** — change the date assigned to the document. Click the date chip to open a date picker, or click "Add date" if none was detected.

When you're satisfied with the classifications, click the **"Create my cap table"** button (or **"Update my cap table"** if you've already imported before). This triggers extraction — the AI reads each document and creates events on the timeline.

While documents are processing, a pulsing **"Processing N docs…"** button appears in the toolbar. Click it to see progress. When classification is ready for review, the button changes to **"N docs classified"** — click to review and confirm.

> **Tip:** You can't upload new documents while a batch is actively processing — the upload area is replaced by a progress animation. Once all documents reach the classification stage (or finish processing), the upload area returns and you can add more files.

### Document Actions

Each document in the processing list has action icons on the right:

- **Cancel** — stops processing on a document that appears stuck (only appears when a document has been in the same processing stage for an unusually long time)
- **Retry** (replay icon) — re-runs processing on a document that failed
- **Download** (down-arrow icon) — opens the original uploaded file in a new tab
- **Dismiss** (X icon) — removes a discarded document from the list (only appears for documents classified as "discarded")

You can also edit document classifications by adding or removing tags directly in the processing list.

## The Timeline

Events are displayed in reverse chronological order (newest first), grouped by date. Each date group shows:

- **Date header** with the formatted date (e.g., "January 15, 2024")
- **Event lines** — each event is a single row showing a narrative description of what happened (e.g., "Issued 100,000 shares of Common Stock to Jane Doe")

### Event Colors

- **Default** — synced events that match the live cap table
- **Green** ("Newly added") — events extracted from recently uploaded documents, not yet committed
- **Orange** (draft) — events that have been edited but not yet committed

### Warning and Review Indicators

Some events may have colored dots on the left:

- **Yellow dot** — the event has a warning or a "review needed" comment (e.g., non-standard amendment terms, unusual values, or other items that need your attention)
- **Red dot** — the event has a potential duplicate warning (e.g., the same issuance appears in multiple uploaded documents)

Events flagged for review have comments parsed from the uploaded documents. You can see these comments when you open the event editor — hover over a comment to reveal options for editing or dismissing it.

### Extraction Confidence

When an event was extracted by AI, the editor shows a **confidence indicator** next to the title:

- **Green** (85%+) — high confidence; the AI is fairly certain about the extracted values
- **Yellow** (65–84%) — review recommended; some values may need verification
- **Red** (below 65%) — low confidence; verify all extracted values against the source document

Hover over the indicator to see the exact percentage. If confidence is below 85%, a banner appears at the top of the editor identifying which fields need attention (e.g., "Verify price per share, quantity against the source document").

### Editing Events

Click any event line to open the **Event Editor** drawer on the right side. The editor shows the full details of that event and lets you:

- **View all fields** — stakeholder, share class, quantity, date, price, vesting terms, etc.
- **Edit fields** — modify any value; changes become draft (unsaved) until you commit
- **Navigate to related entities** — click linked stakeholders, stock classes, or securities to open their editor (with a back button to return)
- **View changes from last synced version** — an expandable section shows what you changed compared to the last committed version (appears only for draft events)
- **Revert** — discard your draft changes and restore the last committed version
- **Delete** — remove an event (with option to delete all versions of that event)

The editor supports many event types including issuances, transfers, cancellations, conversions, exercises, adjustments, acceptances, retractions, vesting events, repurchases, releases, valuations, and financing rounds.

### Adding Events Manually

Click **+ Add** to open the creation drawer, which lets you either upload new documents or manually create new equity events.

## Cap Table AI

If enabled for your account, a sparkle icon (✦) appears next to the **+ Add** button. Click it to open the **Cap Table AI** panel — a chat interface where you can ask questions about your cap table in plain English:

- "Are there any validation issues?"
- "Who are the founders?"
- "How many shares does [stakeholder] own?"

Type `@` in the chat to reference specific events from your timeline for added context.

Conversations are threaded — you can follow up on previous messages. When the AI suggests a change, you can approve or reject it directly in the chat, and provide feedback to improve future suggestions.

---

## Search and Filter

Above the timeline, a search bar lets you find specific events:

- **Text search** — type to filter events by narrative text (matches are highlighted in yellow)
- **Filter by stakeholder** — select a specific person or entity
- **Filter by stock class** — select a specific share class
- **Needs attention** — a counter showing "N need(s) attention" appears when events have warnings; click to filter to just those events

The search bar shows the count of filtered results vs. total (e.g., "Showing 12 of 245 events").

---

## Committing Changes

When you have draft changes (new events from uploads or manual edits), two buttons appear in the toolbar:

### Apply

Click **Apply** to commit all draft events to the live cap table. This makes them permanent and visible in [Summary View](3.4-cap-table-summary-view.md) and [Holder View](3.5-cap-table-holder-view.md).

Before committing, the system checks that all stakeholders have email addresses. If any are missing, you'll see an error message listing the stakeholders that need emails.

### Revert

Click **Revert** to discard all uncommitted drafts. You'll be asked to confirm:

- If this is your first import (no previously synced cap table), reverting deletes all drafts and resets to empty
- If you have a previously synced cap table, reverting restores it to the last committed state
