## Part 1: Open Questions (need answers before going to production)

Three questions came up during the UI walkthrough that we couldn't resolve from code or the UI alone. These directly affect article accuracy.

### Q1: Cap Table — Summary & Holder views (affects article #7)

The Cap Table page has three tabs: **Timeline View** (works), **Summary View** and **Holder View** (both show a "Get Early Access!" form instead of data).

- Is this plan-gated (Free vs One)? Or still in development for everyone?
- If gated, which plan unlocks it?

**Impact:** Article #7 (Upload and Import a Cap Table) mentions all three views. If Summary/Holder are gated or unreleased, we need to either note the requirement or remove the mention.

**Who:** PM or Engineering

### Q2: Publish flow — team-assisted or self-service? (affects article #3)

After a founder checks all 5 boxes and clicks "Yes, I want to publish my deal page," Studio opens an Intercom chat with a pre-filled message ("Hi! I am ready to go live with my Fairmint-Series series.") instead of publishing directly.

- Is publishing always team-assisted? Or only on the Free plan?
- What does the Fairmint team do on their end to complete it?
- Does a ticker symbol form appear for token-based fundraises?

**Impact:** Article #3 (Publish a Fundraise) currently describes this as a team-assisted flow. If there's a self-service path on certain plans, the article needs both paths.

**Who:** PM

### Q3: Shareholder Register PDF — does this feature exist? (affects article #10)

I couldn't find a "Generate Shareholder Register PDF" feature anywhere in Studio (not in Documents, Cap Table, or Tax & Compliance). Article #10 is currently a placeholder.

- Does this feature exist? If so, where in the UI?
- Is it only available when the cap table has data?
- Should we drop this article entirely?

**Impact:** Article #10 is a placeholder right now. Needs either a real writeup or removal from the set.

**Who:** PM or Engineering

## Part 2: Article Review

### How to review

Walk through each article against real Studio. The articles have already been rewritten from a UI walkthrough, but a second set of eyes catches things we missed.

### PM — Accuracy & Completeness

- [ ] **Steps match the UI.** Does each click/action match what the article says?
- [ ] **No missing steps.** Confirmation dialogs, prerequisites, waiting periods?
- [ ] **Fields and options are accurate.** Dropdown options, form fields, toggles?
- [ ] **Terminology matches the UI.** "Fundraise" vs "Series"? "Stakeholders" vs "Investors"?
- [ ] **No phantom features.** Anything described that doesn't exist or is plan-gated?
- [ ] **Deep links work.** Click the `one.fairmint.com` links (need to be logged in).
- [ ] **Business context is right.** SAFE explanations, rollup descriptions, etc.

### Maria — Tone & Clarity

- [ ] **A non-technical founder can follow this.** Read as if you've never used Fairmint.
- [ ] **No jargon.** Flag terms that should be simpler.
- [ ] **Right length.** Target 300-600 words. Too long = won't read. Too short = missing info.
- [ ] **Consistent style.** Numbered steps for actions, bullets for options, **bold** for UI elements.

### Support — Real-World Coverage

- [ ] **Common questions answered.** Would this resolve the questions you get about this topic?
- [ ] **Gotchas covered.** Edge cases or common mistakes to warn about?
- [ ] **Error scenarios.** When something goes wrong, does the article help?
